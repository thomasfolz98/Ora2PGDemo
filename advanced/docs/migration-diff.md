Status: Ora2Pg v24.0, Oracle XE 21c, PostgreSQL 15, `PACKAGE_AS_SCHEMA=1`.

Cost-Report: **Migration level B-5**, 35.80 migration units (~1 PT). 14 Objekte, 0 invalid.

## Teilweise (funktioniert, aber Semantik abweichend)

| Oracle | Postgres-Output | Kommentar |
|---|---|---|
| `DBMS_OUTPUT.PUT_LINE('...')` | `RAISE NOTICE '...'` | Geht nur an Client-Notices, nicht an Serveroutput-Buffer. Fuer Demo-Zwecke ok. |
| Named Exceptions (`e_kunde_nicht_gefunden`, `e_rechnung_nicht_gefunden`, `e_ungueltiger_status`) | `RAISE EXCEPTION 'e_kunde_nicht_gefunden' USING ERRCODE = '50001'` | Ora2Pg erzeugt keine echten benannten Exceptions, sondern einen String + frei gewaehlten SQLSTATE (`50001`..`50003`). Callers, die `WHEN e_kunde_nicht_gefunden THEN` matchen wollten, muessen auf `WHEN SQLSTATE '50001'` umstellen. Beleg: `package.sql:37` |
| `WHEN NO_DATA_FOUND THEN` im `get_kunde` | `EXCEPTION WHEN no_data_found THEN` plus `SELECT ... INTO STRICT` | Postgres braucht `INTO STRICT`, damit `no_data_found` ueberhaupt feuert. ora2pg setzt das automatisch. Beleg: `package.sql:27-35` |
| Package-Lifecycle (Spec + Body + Session-State) | 5 einzelne Functions/Procedures im Schema `pkg_faktura` | Kein Package-State, keine Initialization-Section. Fuer unser Package egal (stateless). |

## Nacharbeit noetig (Patch-Datei)

| Thema | Befund | Fix |
|---|---|---|
| **Return-Typ `NUMBER` -> `bigint`** | `RETURN NUMBER` aus `kunde_umsatz` / `berechne_rechnungssumme` wird von ora2pg (Default-DATA_TYPE-Mapping) auf `bigint` abgebildet. In Postgres: `SELECT pkg_faktura.kunde_umsatz(1)` liefert `881` statt `880.60`, weil bigint implizit rundet. Beleg: `package.sql:44,66` | `migration/output/patches/01_pkg_faktura_numeric.sql` droppt+re-erzeugt beide Functions mit `RETURNS numeric` und `v_summe numeric`. Nach Patch: `880.60` / `420.00`. |
| **Fixed Header-Kommentar `please edit to match PostgreSQL syntax`** | ora2pg schreibt pauschal `-- Oracle package 'PKG_FAKTURA' declaration, please edit to match PostgreSQL syntax.` rein. Beleg: `package.sql:14` | Nur Hinweis, keine Aktion. Der eigentliche Code ist hier valide, die Warnung ist generisch. |
| **`functions.sql` / `procedures.sql` leer** | `-- Nothing found of type FUNCTION` bzw. PROCEDURE. Beleg: `functions.sql:12`, `procedures.sql:12` | Erwartet: alle stored code liegt im Package. Datei bleibt zur Dokumentation. |
| **`sequences.sql` leer** | `-- Nothing found of type SEQUENCE`. Beleg: `sequences.sql:12` | Erwartet: alle Primary Keys sind IDENTITY-Spalten, keine expliziten Sequences im Oracle-Schema. |
| **`-t INDEX` existiert nicht** | ora2pg v24 exportiert Indizes mitsamt `-t TABLE`. Separater Aufruf `-t INDEX` faellt auf die Usage-Seite. | `migrate.sh` ruft `-t INDEX` bewusst nicht mehr auf, Kommentar drin. |
